В файле docker-compose.yml описываем 3 сервиса:

zookeeper: запускаем контейнер и открываем порт для взаимодействия с Kafka и Kafka UI.

kafka: запускаем процесс строго после старта Zookeeper (через depends_on) с параметрами для работы в режиме одного брокера.

сеть и доступы: настраиваем кастомные внешние и внутренние порты (listeners) с протоколами, чтобы Kafka UI мог подключиться внутри сети, а мы — снаружи.

kafka-ui: запускаем после старта Kafka (depends_on: kafka), связываем их по внутренней сети Docker и пробрасываем интерфейс наружу через порт 8080.
